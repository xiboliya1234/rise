# 视频模块实施方案（前后端+数据库）

## 1. 模块目标
- 挂载位置：前端“森罗万象”菜单下新增三个子模块。
- 功能范围：
1. 视频上传（上传本地视频与封面，保存视频信息到数据库）
2. 视频信息管理（查询、编辑、删除）
3. 视频列表查看（分页、在线播放、跳转修改）

## 2. 前端设计（`riseout/riseour`）

### 2.1 路由
- `/video/upload`：视频上传
- `/video/manage`：视频信息管理
- `/video/list`：视频列表查看
- 兼容旧路由：`/video/video` -> 视频列表页

### 2.2 页面文件
- `src/views/video/upload.vue`
- `src/views/video/manage.vue`
- `src/views/video/list.vue`

### 2.3 菜单挂载
- 文件：`src/views/Layout.vue`
- 在“森罗万象”下新增：
1. 视频上传
2. 视频信息管理
3. 视频列表查看

## 3. 后端设计（`new`）

### 3.1 控制器接口（`videoController`）
- `GET /vue/video/page`：分页查询
- `GET /vue/video/{id}`：详情查询
- `POST /vue/video`：新增视频信息
- `PUT /vue/video/{id}`：更新视频信息
- `DELETE /vue/video/{id}`：删除视频
- `POST /vue/video/upload/video`：上传视频文件
- `POST /vue/video/upload/cover`：上传封面文件
- `GET /vue/video/media/{fileName}`：媒体访问（视频/封面）

### 3.2 兼容旧接口
- `GET /vue/video/getVideo` -> 复用 `page`
- `POST /vue/video/saveVideo` -> 复用 `create/update`

### 3.3 服务层规则（`videoService`）
- 分页参数兜底：`pagenum=1`、`pagesize=10`、上限 `100`
- `videocode` 唯一校验（新增/更新都检查）
- 按 ID 查询用于更新/删除前校验

### 3.4 Mapper 查询规则
- 文件：`src/main/resources/mapper/videoinfoMapper.xml`
- 使用 `#{}` + `concat` 防止 SQL 注入
- 支持筛选字段：`videoname`、`videocode`、`videoactor`

## 4. 数据库设计（MySQL：`rise`）

### 4.1 主表
- 表名：`videoinfo`
- 关键字段：
1. `videoname`：视频名称
2. `videocode`：视频代码（唯一）
3. `videourl`：视频访问路径（可在线播放）
4. `videoimg`：封面访问路径
5. `videoactor`：演员信息
6. `videomsg`：简介
7. `videotag`：标签

### 4.2 升级脚本
- 文件：`new/sql/video_module_upgrade.sql`
- 内容包含：
1. 不存在时建表
2. 历史表补 `created_at/updated_at`
3. 索引：`uk_videocode`、`idx_videoname`、`idx_created_at`

## 5. 联调顺序
1. 执行数据库脚本 `new/sql/video_module_upgrade.sql`
2. 启动后端 `new`（端口 `12345`）
3. 启动前端 `riseout/riseour`
4. 先测上传接口，再测新增/分页/编辑/播放闭环

## 6. 验收标准
1. 上传视频和封面成功，数据库有记录
2. 视频列表分页可查，查询条件生效
3. 列表可在线播放视频
4. 管理页可修改视频信息并即时回显
5. 删除后列表与数据库一致
