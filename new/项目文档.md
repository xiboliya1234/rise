## 1. 总体架构

系统已演进为前后端分离：

- 前端（`riseout/riseour`）通过 Axios 访问后端 REST 接口。
- 后端（`new`）主要提供 `/vue/**` 接口，返回统一 `Result`（`code/message/data`）。
- 登录后 token 存在 `localStorage.user.token`，请求拦截器自动加到请求头 `token`。

核心调用链：

1. Vue 页面触发请求。
2. `src/utils/request.js` 注入 token 并请求后端。
3. Spring Boot Controller -> Service -> Dao/Mapper -> MySQL。
4. 返回 `Result` 给前端渲染。

## 2. 目录边界与职责

## 6. 接口与环境一致性问题（已识别）

1. 接口大小写不一致
- 前端同时出现 `/vue/video/getVideo` 与 `/vue/video/getvideo`。
- 后端实际为 `/getVideo`，`getvideo` 可能导致 404。

2. 域名与上下文混用
- `request.js` 基础地址是 `http://localhost:23333/rise`。
- 部分上传/导出使用 `http://localhost:23333/...`（无 `/rise`）。
- OCR 上传又使用 `http://localhost:23333/rise/...`。

3. 外部服务与本地服务并存
- `request3000host.js` 指向 `http://localhost:3000`。
- `ollama.vue` 直接连 `localhost:45454`。
- 部署时需要按环境统一外部地址策略。

4. 路由守卫存在潜在空值问题
- `router.beforeEach` 中 `const user = localStorage.getItem("user")` 后直接访问 `user[0]`。
- 当 `user === null` 时存在运行时错误风险。

## 8. 部署与联调说明（前后端分离）

### 8.1 本地开发

后端（`new`）：

1. 配置数据库与 `riseout.*` 文件路径。
2. 启动后端服务（默认 23333）。

前端（`riseout/riseour`）：

1. `npm install`
2. `npm run serve`
3. 根据当前后端地址调整 `src/utils/request.js` 与上传 action URL。

### 8.2 生产部署

1. 前端执行 `npm run build` 生成 `dist/`。
2. 前端 `dist` 部署到静态站点（Nginx/静态服务器）。
3. 后端作为独立 API 服务部署。
4. 在网关/Nginx 层统一跨域、反向代理与 HTTPS。

### 8.3 Token 与跨域

- token 默认由前端请求拦截器自动注入 header。
- 导出接口采用 URL token 方式，需要与后端拦截器白名单/校验逻辑保持一致。
- 后端 CORS 当前较宽松，生产应按前端域名收敛。

## 9. 联调检查清单

1. API BaseURL 是否统一
- 是否统一使用同一域名、端口、上下文（是否带 `/rise`）。

2. 接口路径大小写是否统一
- 尤其 `getVideo` 与 `getvideo`。

3. 上传与下载域名是否一致
- `el-upload action`、图片回显 URL、导出 URL 是否同源可达。

4. 路由 history 模式是否配置回退
- 前端使用 `history`，服务端需配置 fallback 到 `index.html`。

5. Token 流程是否闭环
- 登录写入 localStorage。
- 请求自动带 token。
- 刷新后布局页可读用户信息。

6. 外部依赖是否可访问
- 高德地图 key、Flowise（3000）、Ollama/SSE 地址等。
